# `data/` — what's tracked vs regenerated

This directory holds dataset artefacts. Most of it is **regenerable from
SimSat / public data sources at runtime** and therefore git-ignored. Only
the small "metadata" and "ground truth" files are committed so the repo is
self-contained at clone time.

| What's committed | Size | Regenerated by | Used by |
|---|---:|---|---|
| `metadata/disaster_m3/*.yaml` (8 files) | ~530 KB | scripts under `scripts/collect_*.py` | DM3 case dropdown in the UI |
| `gt_polygons/*.geojson` (210 files) | 5.1 MB | `scripts/build_scene_catalog.py` (xBD/MCD64A1) + `collect_deforestation.py` (PRODES) | After-map damage / burn / clearing overlay |
| `scene_catalog.yaml` | 19 KB | `scripts/build_scene_catalog.py` | MCD64A1 wildfire catalog |
| `canonical_dataset.yaml` | 195 KB | persisted by the "Save Before/After" UI button | recall canonical Before/After pairs |
| `eval_hf.jsonl` | 6.5 KB | `scripts/eval_wildfire_hf.py` | baseline (HF dataset direct → LFM) |
| `eval_hf_simsat.jsonl` | 15 KB | `scripts/eval_wildfire_hf_simsat.py --use-sentinel-datetime --window-days 1` | baseline (SimSat → preprocess → LFM, recall = 0.933) |

**Total tracked**: ~5.9 MB. Plain text (YAML / GeoJSON / JSONL), no images,
no models, no S2 raster — those are streamed from SimSat at runtime or
pulled from upstream HuggingFace datasets.

## What is NOT tracked (and why)

| Path | Typical size | Why excluded |
|---|---:|---|
| `scenarios/` | 24 GB | SimSat fetch cache (= every Before/After PNG ever fetched). Reproduces itself on first run. |
| `precompute/` | 19 GB | Output of legacy `build_precompute_v2.py`. Not used by current realtime agent path. |
| `curated_pairs/` | 1.4 GB | PNG copies of canonical pairs. Re-creatable by clicking "Save Before/After" or via SimSat fetch. |
| `derived/` | 1.1 GB | Wildfire composite cache. Auto-regenerated. |
| `metadata/disaster_m3/*.csv` | ~180 MB | Heavy xBD CSVs that the YAMLs already extract from. Mirror at <https://disasterm3.s3.amazonaws.com/> if needed. |
| `raw_mcd64a1/`, `wildfire_composites/`, `traces/` | mixed | Per-user / per-experiment artefacts. |
| `eval_*.jsonl` other than the two above | < 100 KB each | Local debug runs (e.g. `eval_hf_simsat_vllm_merged.jsonl` from the merged-ckpt experiment that scored recall = 0). |

## Regeneration recipes

Each `collect_*.py` script writes one yaml under `metadata/disaster_m3/`:

```bash
# From repo root, with .venv active
.venv/bin/python scripts/collect_fireedge_hf.py        # FireEdge GT (300 cases, HuggingFace)
.venv/bin/python scripts/collect_firms_fire.py         # FIRMS active fires      [needs FIRMS_MAP_KEY]
.venv/bin/python scripts/collect_volcanic.py           # GDACS eruptions
.venv/bin/python scripts/collect_ems.py                # Copernicus EMS
.venv/bin/python scripts/collect_deforestation.py      # PRODES Amazon
.venv/bin/python scripts/collect_algal_bloom.py        # algal blooms
.venv/bin/python scripts/collect_negatives_v2.py       # random negatives
.venv/bin/python scripts/collect_hard_negatives.py     # hard negatives
.venv/bin/python scripts/build_scene_catalog.py        # MCD64A1 + xBD scenes  → scene_catalog.yaml + gt_polygons/
```

The committed YAMLs were generated on 2026-04 / 2026-05 (see each file's
`created_at` field). They're stable enough that you should not need to
regenerate just to use the app.

## Eval baseline reproduction

```bash
# wildfire LoRA recall on FireEdge test split, via SimSat (production path)
LFM_WILDFIRE_BASE_URL=http://localhost:8085/v1 \
.venv/bin/python scripts/eval_wildfire_hf_simsat.py \
    --split test --use-sentinel-datetime --window-days 1
# Expected: recall ≈ 0.933 (= what eval_hf_simsat.jsonl records)
```
