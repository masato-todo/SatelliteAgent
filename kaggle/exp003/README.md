## Stage layout (regenerated by `./upload.sh`)

```
stage/
├── curated_pairs/<case_id>/{before,after}.png + meta.yaml   # 482 dirs, 8.3GB
├── gt_polygons/<case_id>.geojson                            # MCD64A1+PRODES only, 5.6MB
├── canonical_dataset.yaml                                   # master 482-entry list
├── scene_catalog.yaml                                       # MCD64A1 catalog
├── traces/<scene_id>__YYYYMMDDTHHMMSSZ.yaml                 # Phase 3 Gemini ReAct
└── dataset-metadata.json
```

After `kaggle datasets create --dir-mode zip`, the depth-1 directories
become 3 zips: `curated_pairs.zip` (8.3GB), `gt_polygons.zip` (5.6MB),
`traces.zip` (288KB). Top-level yamls are uploaded as plain files.

## Re-upload

```bash
cd kaggle/exp003
./upload.sh                          # initial → kaggle datasets create
UPDATE=1 MSG="..." ./upload.sh       # subsequent → kaggle datasets version
```

`upload.sh` regenerates `stage/` from `data/` every time. Update the
local `data/` (re-run `auto_fill_pairs.py` etc.) and re-run.

## Pairing with precompute-v3

The id of every case in `canonical_dataset.yaml` matches the zip filename
in `<KAGGLE_USER>/satelliteagent-precompute-v3` (= [kaggle/exp004](../exp004/)).
Use raw-v3 for image inspection / VLM input; use precompute-v3 for
offline tool replay.
